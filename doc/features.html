Features

Nicer user interaction

Bootstrap Darkmode Toggle select the color scheme to render in function of:

  • Site cookie (if enabled). To keep the user preference in the site a cookie save the prefered color scheme each time the color scheme is changed.
  • User preferred color scheme. If the browser is set to preffered darkmode, darkmode will be rendered.
  • Data-Atributes and API options.

State

Bootstrap Darkmode initial state can be set by data-state attribute.
The site cookie (if enabled) or the user preferred color scheme dark is prioritary.

Custom Root Element

Bootstrap Darkmode root element can be set by data-root attribute.

Custom Label

Bootstrap Darkmode light and dark label can be set by data-lightlabel and data-darklabel attributes.

Custom Color Scheme

Bootstrap Darkmode light and dark color scheme can be custumized by data-lightColorMode and data-darkColorMode attributes.
Refer to Bootstrap Color Mode Documentation for help.

Prefered Color Scheme in Cookie

Bootstrap Darkmode Toggle can save user prefered color schema in a cookie. To enable this feature add allowCookie data attribute or API option.

As per the GDPR and ePrivacy Directive, a website must ask its users’ consent to use cookies that are not necessary for accessing the website’s functionality. These cookies need consent because they collect user data for their purposes. According to the law, collecting data without users’ consent is unlawful.

The employed cookie is a User interface (UI) customization cookie that do not requiere a explicit authorization. However the default value is set to false in order to allow requesting cookie consent if wanted.

If consent is provided, the cookie usage can be enabled with allowCookie method or disabled with denyCookie method.
